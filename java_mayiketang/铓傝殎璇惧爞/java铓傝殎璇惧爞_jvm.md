1.jvm内存调优和结构原理
 (1)java内存结构(jvm)与java内存模型(多线程jmm)
    java内存模型jmm是关于内存可见性
    java内存结构是指的jvm
    
 (2)jav内存结构分为
    1.方法区：static关键修饰、常量信息class文件加载的时候，就会被初始化。所有线程会被共享。注意线程安全问题；
      调优问题：web开发定义太多常量太多不好
    2.堆：创建对象、new创建的数组 存放堆内存---调优策略堆。堆所有线程会共享
    3.栈：1.由三部分组成局部变量、操作数据栈和帧数据区。
          2.每个线程私有，互不共享，栈不会产生线程安全问题；定义局部变量，方法参数，java的方法调用，返回值等；栈代码运行完毕，自动释放内存。
          3.本地方法栈：用于本地方法调用
    4.PC寄存器 每个线程私有，java虚拟机会为每个线程创建PC寄存器
    5.执行器：执行字节码文件

    1.基本类型和对象的引用都放在栈中，new出的对象和数组放在堆中
    2.名称；特征；作用；配置参数
    程序计数器；占用内存小，线程私有，生命周期与线程相同    大致为字节码行号指示器；大致为字节码行号指示器
    虚拟机栈；线程私有，生命周期与线程相同，使用连续的内存空间；Java 方法执行的内存模型，存储局部变量表、操作栈、动态链接、方法出口等信息；-Xss
    java堆；线程共享，生命周期与虚拟机相同，可以不使用连续的内存地址；保存对象实例，所有对象实例（包括数组）都要在堆上分配；-Xms-Xsx-Xmn
    方法区；线程共享，生命周期与虚拟机相同，可以不使用连续的内存地址；存储已被虚拟机加载的类信息、常量、静态变量、即时编译器编译后的代码等数据；-XX:PermSize:16M-XX:MaxPermSize64M
    运行时常量池；方法区的一部分，具有动态性

 (3)新生代与老年代
    什么是堆，new出来的对象都会放到堆内存中
    堆分为两个区：新生代、老年代
    新生代分为三个区：eden(占新生代1/2的空间)、s0(1/4的空间)、s1(1/4的空间)，s0和s1也被称为from区和to区
    刚创建的对象会方到eden区中
    调优：在web开发中尽量少时间常量信息，因生命周期是永久的会占用大量的内存空间
        尽量减少垃圾回收机制的次数
        新生代回收次数比老年代多

 (4)堆内存参数配置
     -XX:+PrintGC      每次触发GC的时候打印相关日志
     -XX:+UseSerialGC      串行回收
     -XX:+PrintGCDetails  更详细的GC日志
     -Xms               堆初始值 此值可以设置与-Xmx相同，以避免每次垃圾回收完成后JVM重新分配内存
     -Xmx               堆最大可用值
     -Xmn               新生代堆最大可用值  一般设为整个堆的1/3到1/4左右
     -XX:SurvivorRatio     用来设置新生代中eden空间和from/to空间的比例. 例如:-XX:SurvivorRatio=2,表示eden区是s0区或者s1区的2倍
     含以-XX:SurvivorRatio=eden/from=den/to
     -XX:newSize：表示新生代初始内存的大小，应该小于-Xms的值；
     -XX:MaxNewSize：表示新生代可被分配的内存的最大上限；当然这个值应该小于-Xmx的值；
     -XX:PermSize：表示非堆区初始内存分配大小（方法区）
     -XX:MaxPermSize：表示对非堆区分配的内存的最大上限（方法区）。
     -Xss5m  表示栈的最大深度
     堆初始值越小，垃圾回收机制次数越多
   1.堆初始化值与内存最大值要保持一致，减少垃圾回收次数；为了防止垃圾收集器在最小、最大之间收缩堆而产生额外的时间，我们通常把最大、最小设置为相同的值。
   2.设置新生代与老年代的比例，新生代越小，新生代的回收几率越高
   -XX:NewRatio=2 表示新生代与老年大的比例是1:2，也就是老年代是新生代的2倍
   参数: -Xms20m -Xmx20m -Xmn1m -XX:SurvivorRatio=2 -XX:+PrintGCDetails -XX:+UseSerialGC

 (5)堆溢出
     错误原因: java.lang.OutOfMemoryError: Java heap space 堆内存溢出
     解决办法:设置堆内存大小 -Xms1m –Xmx10m -XX:+HeapDumpOnOutOfMemoryError

 (5)栈溢出
     错误原因: java.lang.StackOverflowError  栈内存溢出
    栈溢出产生于方法递归调用，遍历不会，但是循环方法里面产生递归调用，也会发生栈溢出
    栈会有一个深度，可以通过-Xss5m 设置最大调用深度

 (6)Tomcat内存溢出在catalina.sh 修改JVM堆内存大小
    JAVA_OPTS="-server -Xms800m -Xmx800m -XX:PermSize=256m -XX:MaxPermSize=512m -XX:MaxNewSize=512m"
    -XX:PermSize=256m JVM初始分配的非堆内存
    -XX:MaxPermSize=128M JVM最大允许分配的非堆内存，按需分配
    -XX:MaxnewSize：表示新生代可被分配的内存的最大上限；当然这个值应该小于-Xmx的值；

(7)jvm调优总结
    GC时间较短，GC次数较少，GC的周期足够的长
    前两个的条件是相克的，也就是如果要时间较短则堆内存设置较小，如果次数较少则需要足够大的堆内存；因此只能取平衡。
    新生代和老年代设置多少更加合理？
     1.如果临时变量较多，则将应该设置较大的新生代，如果较多的持久对象，则应该设置较大老年代，但是一般程序很难有这样的明显特征；
     2.以GC次数少的原则，让老年代尽量缓存常用对象；
     3.JVM的默认比例1：2，通过观察应用一段时间，看其他在峰值时年老代会占多少内存；
     4.在不影响GC的情况下，根据实际情况加大新生代，比如控制在1：1，但是要老年代至少预留1/3的空间。

2.垃圾回收机制
  (1)什么是垃圾回收机制
    垃圾回收机制不定时，向堆内存清理不可达对象；不可达对象就是没有被任何东西引用

  (2)垃圾回收机制的算法有几种？
    引用计数法  --每个对象有15次机会，每次垃圾回收时进行检查是否为不可达对象，如果是不可达就减一，直到减到零后对该对象进行回收
    标记清除  ---碎片化
    标记压缩清除 ---对标记的垃圾打包到一起
    复制算法  --s0  s1
    分代算法  ---分为新生代和老年代
    finalize()方法是Object中的方法，是在将对象进行垃圾回收之前执行的，如果要看这个对象是否被垃圾回收，可以重写该方法
    System.gc(); --手动GC，这个只是通知GC进行回收，具体回收还得看GC

  (3)内存溢出和内除泄露
    内存溢出：需要4G内存  但是只支持3G内存（内部不够）
    内存泄露：定义很多的静态变量，垃圾回收机制不会回收，这个对象没有引用，报错泄露。（存在浪费内存）

  (4)引用计数法
    引用计数法，每个对象添加一个引用计数器，当计数器为0时，GC时进行回收，
    默认15次，gc回收时，对象不可达减1，可达加1；如果加到15就会有可能进入到s0区s1区老年代。
    优点：可以很快的执行，不会长时间打断程序。
    缺点：无法检查循环引用，如果父类引用子类，子类引用父类，这样他们永远都不会为0，每次加减很让费内存。

  (5)复制算法
    复制算法用于新生代，新生代分为eden区s0区s1区，刚创建的对象是放在eden区，经过20次左右的GC后会进入s0区或者s1区，
    然后再经历过大概15次左右就会进入老年代，老年代是回收次数少，不是不回收。
    在新生代中，复制算法是将可达对象复制到s0区或者s1区中，然后再将s0区或者s1区的整个区都清空
    优点：不会产生内存碎片
    缺点：可使用内存降低一半

  (6)标记清除算法和标记压缩算法
    标记清除算法：为每个对象做一个标记，0表示可达，1表示不可达；如果对象经常不被使用将对象修改为1
    缺点：碎片化，不是连续性
    标记压缩算法：在标记清除算法的基础上，将不可达对象进行整理，将不可达对象全部放在最前面，然后一起清除
    优点：连续性，无碎片
    老年代使用标记压缩算法

  (7)分代收集算法
    java虚拟机中一般是把内存分为新生x代和老年代，新创建的对象一般先分配到新生代，经过几次GC后仍然存活的对象移动到老年代；
    当大对象在新生代无法找到足够的连续内存时，直接在老年代中创建。
    新生代回收频率高，但时间短；而老年代回收频率低，但时间长；所以得减少老年代的GC。

  (8)分代算法
    分为新生代和老年代（使用标记压缩算法）
    新生代：eden区（使用的是引用计数法）、S0区和S1区（使用的是复制算法）
    垃圾回收机制如果频繁执行为什么会影响效率？
       消耗内存；会出现卡顿，因为GC是会暂停所有线程，暂停线程是为了更好的回收垃圾，避免产生新的垃圾没有回收干净。
       
  (9)垃圾收集器
    串行回收：单线程回收垃圾，回收期间暂停所有线程；-XX:+UseSerialGC命令行可选项强制指定，jvm默认使用串行回收
    并行回收：多线程回收垃圾；-XX:+UseParallelOldGC 进行设置；-XX:+ParallelCThread也可以设置垃圾收集时的线程教量。
    
  (10)调优总结
    初始堆值和最大堆内存内存越大，吞吐量就越高且GC次数越少
    最好使用并行收集器,因为并行收集器速度比串行吞吐量高，速度快。
    设置堆内存新生代的比例和老年代的比例最好为1:2或者1:3。
    减少GC对老年代的回收。
