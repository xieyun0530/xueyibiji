1.事务隔离级别
    事务隔离级别                          脏读              不可重复读           幻读
    读未提交（read-uncommitted）          是                   是                 是
    不可重复读（read-committed）          否                   是                 是
    可重复读（repeatable-read）           否                   否                 否(MySql解决了幻读，使用的是间隙锁解决的)
    串行化（serializable）                否                   否                 否
    (1)脏读
        事务A读取了事务B未提交的数据，那么A读取到的是脏数据，因为A事务有可能反射回滚。
    (2)不可重复度
        事务A多次读取同一条数据，事务B在事务A读取一次数据后，将A事务读取的数据做了修改，导致事务A多次读取的数据不一致。
    (3)幻读
        事务A多次读取的数据条数不一致，事务B在事务A读取一次数据后，将A事务读取到的某一条数据删除，导致事务A多次数据的
        数据条数不一致。
    (4)间隙锁
        在A会话中：insert into t_lock_1 values(10),(11),(13),(20),(40);
                   begin
                   select * from t_lock_1 where a <= 13 for update;
        在B会话中：insert into t_lock_1 values(19) 阻塞；
        间隙锁会扫描到当前值（13）的下一个值（20）,并把这些数据全部加锁，就是会把13到20之间的数据上锁；
